# Веб-платформа для управления образовательными задачами

Краткое описание

Веб-платформа для руководителя образовательной программы, которая повышает прозрачность процессов, упрощает коммуникацию и обеспечивает контроль за выполнением задач преподавательским составом.

Цель проекта

- Создать единое пространство для ведения образовательной деятельности.
- Повысить ответственность участников процесса.
- Устранить хаос в коммуникации между руководителем и преподавателями.

Ролевая модель

- Руководитель образовательной программы
  - Инициирует и формирует задачи.
  - Назначает ответственных (преподавателей).
  - Устанавливает первичные сроки и контролирует ход выполнения.

- Ответственный (Преподаватель)
  - Исполняет назначенные задачи.
  - В процессе работы может корректировать сроки и обновлять статус выполнения.

Возможности

1) Управление задачами
   - Создание задачи с заголовком и описанием (например, "Организация практики").
   - Назначение конкретного исполнителя из списка преподавателей.
   - Установка первичного срока выполнения.
   - Возможность ответственному корректировать установленные сроки по мере работы.
   - Отображение текущего статуса и прогресса по задаче.

2) Обзор и контроль
   - Общий обзор, где видно, кто над чем работает в текущий момент.
   - Просмотр задач в работе, назначенных исполнителей и их загрузки.
   - Актуальные сроки и статусы задач для предотвращения дублирования усилий.

3) Прозрачность и ответственность
   - Историзация изменений сроков и статусов (для понимания динамики работы).
   - Четкое разграничение ролей: кто создал задачу и кто её выполняет.

Жизненный цикл задачи

- Создана (инициирована руководителем)
- Назначена (определён ответственный)
- В работе (ответственный обновляет прогресс и может корректировать срок)
- Завершена (задача выполнена и отмечена как завершённая)
- (При просрочке) Просрочена - отражается в обзоре для оперативного вмешательства

Сценарии использования

- Руководитель создает задачу "Организация практики", назначает преподавателя и ставит дедлайн «Февраль».
- Преподаватель начинает работу, в процессе переносит срок на более поздний и отмечает текущий прогресс.
- Руководитель отслеживает изменения и общий дашборд, видит загрузку преподавателей и предотвращает дублирование работ.

Польза для образовательной программы

- Формализация процессов: все задачи и сроки в одном месте.
- Повышение ответственности исполнителей благодаря прозрачности.
- Сокращение хаоса в коммуникации между руководством и преподавателями.
- Улучшенное планирование за счёт видимости загрузки персонала.
